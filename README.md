A relational database project that demonstrates the design and implementation of a Student & Course Management System using MySQL. The system efficiently manages students, instructors, courses, enrollments, and grades while providing meaningful academic insights through SQL queries.

⸻

 Project Overview

This project was developed to simulate an academic database that supports student record management, course enrollment, performance tracking, and reporting. It leverages SQL to organize academic data, maintain data integrity, and generate analytical reports that support informed academic decision-making.

⸻

 Features

* Relational database design using SQL
* Student and instructor management
* Course and enrollment tracking
* Grade management
* Student performance analysis
* Academic reporting
* Course enrollment analysis
* Instructor performance evaluation
* Data integrity through relational constraints
* Business intelligence using SQL queries

⸻

 Technologies Used

* MySQL – Database design and SQL querying
* MySQL Workbench – Database development and query execution
* Microsoft Word – Project documentation
* Google Meet – Team collaboration and project coordination

⸻

 Database Schema

The database consists of the following tables:

* Students
* Instructors
* Courses
* Enrollments
* Grades

The schema uses Primary Keys, Foreign Keys, and relational constraints to ensure data integrity and establish accurate relationships between entities.

SQL Analysis Performed

The project includes SQL queries to analyze:

* Student academic performance
* Average scores by course
* Student enrollment records
* Course enrollment statistics
* Instructor performance
* Students requiring academic support
* Best-performing courses
* Academic summary reports

Queries utilize:

* JOINs
* Aggregate Functions
* GROUP BY
* ORDER BY
* HAVING
* Subqueries
* Filtering
